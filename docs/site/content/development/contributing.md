# Contributing to Pixie Documentation

Thank you for your interest in contributing to the Pixie documentation! This guide explains how to contribute to the project and maintain the quality of the documentation.

## Getting Started

1. Fork the repository on GitHub
2. Create a new branch for your changes
3. Make your changes following the guidelines below
4. Submit a pull request with a clear description of your changes

## Documentation Guidelines

### Writing Style

- Use clear and concise language
- Write in an active voice when possible
- Use consistent terminology throughout the documentation
- Organize content logically with appropriate headings

### Content Structure

1. Begin with a brief introduction of the topic
2. Provide step-by-step instructions where applicable
3. Include code examples when relevant
4. Add explanations for any complex concepts
5. End with a summary or next steps when appropriate

### Markdown Guidelines

When contributing to the documentation, please follow these Markdown conventions:

- Use ATX-style headings (`#`, `##`, `###`) with a space after the hash
- Use backticks (`) for inline code
- Use triple backticks (```) for code blocks
- Include syntax highlighting by specifying the language after the opening backticks
- Use relative links when linking to other documentation pages

## Technical Requirements

### Front Matter

All documentation pages should include appropriate front matter with the following fields:

```yaml
---
title: "Page Title"
date: 2025-12-08
---
```

### Hugo Features

- Use Hugo's shortcode features where appropriate
- Include alt text for all images
- Link to related content using Hugo's built-in linking features

## Submitting Changes

1. Ensure your changes are formatted properly with Markdown
2. Verify all links are working correctly
3. Test the site locally to make sure your changes look correct
4. Submit a pull request with a descriptive title and detailed description of your changes

## Code of Conduct

Please follow the project's code of conduct when contributing. Be respectful and constructive in all interactions.
